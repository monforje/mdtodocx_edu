# Как писать Markdown для mdtodocx

## Запуск

```bash
mdtodocx input.md output.docx
```

---

## Заголовки

### Центрированные (Введение, Заключение, Список литературы)

```markdown
# Введение {.centered}
# Заключение {.centered}
# Список литературы {.centered}
```

Класс `{.centered}` ставится **после** текста заголовка.

### Нумерованные разделы

```markdown
# 1 Название раздела
## 1.1 Подраздел
### 1.1.1 Подподраздел
```

Разрыв страницы вставляется автоматически перед каждым `#` (кроме первого).

---

## Текст

Обычные абзацы — просто текст, разделённый пустой строкой:

```markdown
Первый абзац.

Второй абзац.
```

---

## Разрыв страницы

```markdown
\newpage
```

---

## Списки

### Маркированный

```markdown
- первый пункт;
- второй пункт;
- третий пункт.
```

Все пункты пишутся нижним регистром и заканчиваются `;`, последний — `.`.

### Нумерованный

```markdown
1. Первый пункт.
2. Второй пункт.
3. Третий пункт.
```

---

## Таблицы

Подпись ставится **после** таблицы через `: `:

```markdown
| Параметр | Значение | Описание |
| -------- | -------- | -------- |
| Скорость | 100 Мбит | Полоса   |

: Таблица 1 - Характеристики сети
```

---

## Изображения

```markdown
![Рисунок 1 - Название](images/pic.png){width=80% fig-align="center"}
```

---

## Выделение текста

```markdown
**жирный**
*курсив*
```

---

## Тире

Используй `-`, не `—`. Фильтр заменяет `—` на `-` автоматически.

---

## Список литературы

```markdown
# Список литературы {.centered}

1. Иванов И.И. Название. - М.: Изд-во, 2023. - 200 с.
2. Петров П.П. Статья // Журнал. - 2022. - № 3. - С. 15-20.
```

---

## Типичная структура (пример)

```markdown
# Введение {.centered}

Текст введения.

# 1 Первый раздел

## 1.1 Подраздел

Текст.

### 1.1.1 Подподраздел

Текст.

![Рисунок 1 - Описание](images/pic.png){width=80% fig-align="center"}

| А | Б |
|---|---|
| 1 | 2 |

: Таблица 1 - Название

# Заключение {.centered}

Текст заключения.

# Список литературы {.centered}

1. Источник.
```
