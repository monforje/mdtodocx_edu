# Правила оформления Markdown для академических работ

## Запуск

```bash
python -m mdtodocx input.md output.docx
```

С опциональным шаблоном:

```bash
python -m mdtodocx input.md output.docx --ref docx/ref.docx
```

Сборка бинарника:

```bash
uv run pyinstaller mdtodocx.spec --clean --distpath dist --workpath build
```

Запуск бинарника (шаблон `ref.docx` встроен):

```bash
./dist/mdtodocx input.md output.docx
```

**Пайплайн обработки:**
1. Pandoc с Lua-фильтром `filters/academic.lua` конвертирует Markdown → DOCX
2. Python-модуль `postprocess.py` исправляет таблицы, списки и нумерацию в XML
3. Результат — финальный DOCX с академическим оформлением

## Структура документа

### Заголовки по центру (Введение, Заключение, Список литературы)

Добавь класс `{.centered}` **после** текста заголовка — получишь стиль **center_heading** (по центру, жирный, 14pt):

```markdown
# Введение {.centered}

# Заключение {.centered}

# Список литературы {.centered}
```

**Важно:** `{.centered}` ставится ПОСЛЕ текста заголовка, не перед ним!

### Нумерованные разделы

Обычный `#` без класса → стиль **heading_1** (отступ 1.25, жирный, по ширине):

```markdown
# 1 Анализ предметной области

## 1.1 Обзор существующих решений

### 1.1.1 Сравнительный анализ
```

Разрывы страницы автоматически вставляются перед каждым заголовком уровня 1 (кроме первого).

---

## Основной текст

Обычные абзацы автоматически получают стиль **main_text** (14pt, отступ 1.25, интервал 1.5, по ширине).

```markdown
Это обычный абзац. Он будет отформатирован с отступом первой строки 1.25 см,
межстрочным интервалом 1.5 и выравниванием по ширине.

Следующий абзац. Пустая строка между абзацами обязательна.
```

---

## Разрыв страницы

Автоматический разрыв перед каждым заголовком уровня 1. При необходимости дополнительного разрыва:

```markdown
\newpage
```

или:

```markdown
::: {.pagebreak}
:::
```

## Списки

### Маркированный список

```markdown
- Первый пункт;
- Второй пункт;
- Третий пункт.
```

- Шрифт 14pt, отступ 1.25, интервал 1.5, по ширине
- Все пункты заканчиваются `;`, кроме последнего — `.`
- Маркер: дефис (`-`)
- Стиль: **marker_list**

### Нумерованный список

```markdown
1. Первый пункт.
2. Второй пункт.
3. Третий пункт.
```

- Шрифт 14pt, отступ 1.25, интервал 1.5, по ширине
- Все пункты заканчиваются `.`
- Стиль: **num_list**

## Таблицы

Стандартный Markdown-синтаксис. Подпись ставится **после** таблицы через `: `:

```markdown
| Параметр    | Значение | Описание          |
| ----------- | -------- | ----------------- |
| Скорость    | 100 Мбит | Пропускная полоса |
| Задержка    | 5 мс     | Время отклика     |

: Таблица 1 - Характеристики сети
```

- Таблица автоматически растягивается по ширине окна (100%)
- Заголовки таблицы — **жирным** шрифтом, 12pt
- Текст таблицы — 12pt
- Подпись таблицы — шрифт 14pt с разрежённым интервалом (letter-spacing 20)
- Стиль подписи: **table_heading**

---

## Тире

Всегда используйте одинарное дефис-тире (`-`), а не двойное (`—`):

```markdown
Правильно: существенный недостаток - единую точку отказа
Неправильно: существенный недостаток — единую точку отказа
```

Фильтр автоматически заменяет `—` на `-`.

## Изображения

```markdown
![Рисунок 1 - Архитектура системы](images/arch.png){width=80%}
```

Для изображения по центру с подписью:

```markdown
![Рисунок 1 - Архитектура системы](images/arch.png){width=80% fig-align="center"}
```

- Изображение автоматически центрировано при наличии `fig-align="center"`
- Подпись к изображению — по центру, 12pt (стиль `image_heading`)
- Подпись размещается **под** изображением

## Список литературы

Оформляется как нумерованный список после заголовка `# Список литературы {.centered}`:

```markdown
# Список литературы {.centered}

1. Иванов И.И. Название книги. - М.: Издательство, 2023. - 200 с.
2. Петров П.П. Статья в журнале // Журнал. - 2022. - № 3. - С. 15-20.
3. Сидоров С.С. [Электронный ресурс]. - URL: https://example.com (дата обращения: 01.01.2024).
```

## Стили (для справки)

| Стиль | Описание | Применение |
|-------|----------|------------|
| `center_heading` | По центру, жирный, 14pt | Введение, Заключение, Список литературы |
| `heading_1` | Отступ 1.25, жирный, по ширине, 14pt | Основные разделы |
| `heading_2` | Подразделы | `## 1.1` |
| `heading_3` | Подподразделы | `### 1.1.1` |
| `main_text` | 14pt, отступ 1.25, интервал 1.5, по ширине | Обычные абзацы, изображения (с центрированием) |
| `marker_list` | 14pt, отступ 1.25, интервал 1.5 | Маркированные списки |
| `num_list` | 14pt, отступ 1.25, интервал 1.5 | Нумерованные списки |
| `table_heading` | 14pt, letter-spacing 20 | Подписи таблиц |
| `image_heading` | 12pt, по центру | Подписи изображений |

## Типичная структура файла

```markdown
# Введение {.centered}

Текст введения...

# 1 Название первого раздела

## 1.1 Подраздел

Текст подраздела...

### 1.1.1 Подподраздел

Текст...

![Рисунок 1 - Название рисунка](images/pic.png){width=80% fig-align="center"}

# 2 Название второго раздела

Текст...

| Параметр | Значение |
|----------|----------|
| А        | 1        |

: Таблица 1 - Название таблицы

# Заключение {.centered}

Текст заключения...

# Список литературы {.centered}

1. Источник первый...
2. Источник второй...
```

Разрывы страниц вставляются автоматически перед каждым заголовком уровня 1.
